# Contributing to Lord of Mysteries TRPG System

Lord of Mysteries TRPG 시스템에 기여해주셔서 감사합니다!

## 기여 방법

### 버그 리포트

버그를 발견하셨다면:

1. [Issues](https://github.com/skkinlive/lord-of-mysteries/issues)에서 이미 보고된 버그인지 확인
2. 새 Issue 생성
3. 다음 정보 포함:
   - 버그 설명
   - 재현 방법
   - 예상 동작
   - 실제 동작
   - 스크린샷 (가능한 경우)
   - Foundry VTT 버전
   - 시스템 버전

### 기능 제안

새로운 기능을 제안하고 싶으시다면:

1. [Issues](https://github.com/skkinlive/lord-of-mysteries/issues)에서 유사한 제안이 있는지 확인
2. 새 Issue 생성 (제목에 [Feature Request] 추가)
3. 다음 정보 포함:
   - 기능 설명
   - 사용 사례
   - 기대 효과

### Pull Request

코드를 직접 기여하고 싶으시다면:

1. 리포지토리 Fork
2. 새 브랜치 생성 (`git checkout -b feature/AmazingFeature`)
3. 변경사항 커밋 (`git commit -m 'Add some AmazingFeature'`)
4. 브랜치에 Push (`git push origin feature/AmazingFeature`)
5. Pull Request 생성

## 코딩 스타일

- JavaScript: ES6+ 문법 사용
- 들여쓰기: 2 spaces
- 주석: 한국어 또는 영어
- 파일명: kebab-case
- 클래스명: PascalCase
- 함수명: camelCase

## 커밋 메시지

- 명확하고 간결하게 작성
- 한국어 또는 영어 사용
- 예시:
  - `Add: 새로운 기능 추가`
  - `Fix: 버그 수정`
  - `Update: 기존 기능 업데이트`
  - `Docs: 문서 수정`

## 테스트

Pull Request 전에 다음을 확인해주세요:

- [ ] Foundry VTT에서 시스템이 정상 로드되는지 확인
- [ ] 새로운 기능이 의도대로 작동하는지 테스트
- [ ] 기존 기능에 영향을 주지 않는지 확인
- [ ] 콘솔 에러가 없는지 확인

## 질문

궁금한 점이 있다면 [Issues](https://github.com/skkinlive/lord-of-mysteries/issues)에서 질문해주세요!
